# 115网盘目录选择功能设计规范

## 1. 目标描述
为 STRM 管理页面的 115网盘“目标目录 CID”输入框增加一个可视化、交互式的文件夹选择器（Folder Picker），允许用户直接浏览 115网盘中的目录树并进行选择，避免手动复制和粘贴 ID。

## 2. 交互与用户体验
* **触发机制**：在目标目录 CID 输入框旁边新增“选择目录”按钮。
* **初始定位（智能加载）**：点击打开弹窗时，如果输入框内有合法的 CID，则优先加载该 CID 对应的目录。如果加载失败或输入框为空，则退回加载根目录（`0`）。
* **选择器展示内容**：
  * **仅展示文件夹**：出于清爽和性能考虑，过滤掉实际文件，列表中仅显示子目录。
  * **弹窗风格**：采用与整体系统风格匹配的高级毛玻璃暗黑风格 (Glassmorphism)。
* **导航操作**：
  * 顶部显示“返回上级”按钮。
  * 点击任意文件夹进入该目录的下一级。
  * 选中的目录具有高亮选中状态。
* **确认逻辑**：点击弹窗右下角的“确认选择”按钮后，弹窗关闭，输入框被自动填入对应选中的目录 ID。
* **异常提示**：若未登录或 Cookie 失效，弹窗内会展示对应错误提示并引导用户去登录。

## 3. 技术设计

### 3.1 后端接口 (Backend API)
* **`app/core/cloud115/client.py`**:
  * 新增 `list_dirs(self, dir_id: str = '0')` 方法。
  * 使用较大的 `limit` 批量调用底层 API。
  * 过滤响应数据，移除包含 `"fid"` 的文件项目。
  * 提取目录 `cid`、名称 `n` 和父级 ID `pid`。
  * 返回结构：`{"current_dir": "名称", "parent_id": "父级ID", "dirs": [...] }`。

* **`app/api/cloud115.py`**:
  * 新增路由：`GET /api/v1/115/dirs`。
  * 接收 `dir_id` 参数。
  * 在内部调用 `client_115.list_dirs(dir_id)` 并在发生错误时返回适当的 HTTP 状态码。

### 3.2 前端实现 (Frontend)
* **`app/web/templates/strm.html`**:
  * **HTML**: 在输入框所在 `div` 中追加“选择目录”按钮。
  * **Modal HTML**: 添加弹窗 HTML 结构，包括遮罩层、弹窗主体、头部、中间列表容器、底部操作区。
  * **CSS**: 编写 Modal 的特定样式，集成 `style.css` 既有的 `.glass-panel` 等变量。
  * **JS 逻辑**: 
    * `openFolderPicker()`: 获取当前输入框 CID 并发起加载请求。
    * `loadFolder(dirId)`: 获取数据，并渲染列表、维护当前路径层级。
    * 选中的 CID 记录在全局变量或特定 DOM 属性中。
    * 确认按钮事件、关闭按钮事件。

## 4. 容错与边界情况
* 如果用户网络断开或请求超时，界面应给出失败提示，而不是无限 loading。
* 如果用户输入了非法的目标目录 CID 进行加载，需退回 `0` (根目录) 再次尝试。
* 若目录下只有文件没有子文件夹，界面需展示“该目录下暂无子文件夹”的占位空状态。

## 5. 设计验收标准
1. API 必须成功过滤掉文件，仅返回文件夹信息。
2. 前端选择弹窗不影响原有布局。
3. 可多层级深入和返回上级，不会导致导航死循环或报错。
4. 无论 115 目录多深，都可通过“返回上级”返回到根目录（或由后端 `parent_id` 自动定位）。
